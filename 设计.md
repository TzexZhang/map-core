## 1. 项目概述

### 1.1 背景与目标

**MapCore** 是一个基于 OpenLayers（2D）与 Cesium（3D）封装的通用地图能力 SDK。

目标：

- 屏蔽 OpenLayers 与 Cesium 的底层 API 差异，对外暴露**统一 TypeScript 接口**
- 纯 TS 实现，**不绑定任何前端框架**（React / Vue / Angular 均可使用）
- 支持**多端部署**：传统 Web、前端框架、Qt WebEngine、移动 WebView、嵌入式终端
- 数据源统一管理，支持 HTTP / HTTPS / WebSocket，预留调试与联调空间
- **内网友好**：所有外部依赖均可替换为内网地址，无强制 CDN 依赖
- 插件化架构，业务方可无侵入扩展自定义能力

### 1.2 核心能力矩阵

| 能力域     | 功能点                                             | 优先级 |
| ---------- | -------------------------------------------------- | ------ |
| 地图渲染   | 瓦片加载、矢量渲染、栅格叠加、三维地形             | P0     |
| 数据源管理 | HTTP/HTTPS/WebSocket/本地/Mock 数据                | P0     |
| 图层系统   | 图层增删改查、可见性、透明度、层叠顺序             | P0     |
| 事件系统   | 地图事件、数据事件、业务事件、跨端通信             | P0     |
| 多端适配   | UMD/ESM、WebComponent、iframe Bridge、NativeBridge | P1     |
| 调试能力   | 日志分级、性能监控、数据回放、Mock 注入            | P1     |
| 插件机制   | 注册/卸载、依赖声明、生命周期钩子                  | P1     |
| 内网部署   | 离线瓦片、内网代理、资源本地化                     | P2     |

### 1.3 技术选型说明

| 项目     | 选型              | 说明                                |
| -------- | ----------------- | ----------------------------------- |
| 语言     | TypeScript 5.x    | 严格模式，完整类型推导              |

| 3D 引擎  | Cesium 1.x        | 三维地球，CZML/3DTiles/地形         |
| 打包工具 | Rollup / Vite     | 产出 ESM + UMD 双格式               |

| 测试     | Vitest            | 单元测试，覆盖核心模块              |
| 包管理   | pnpm workspace    | Monorepo，模块独立发布              |

## 2. 设计原则

### 2.1 接口稳定优先

- 公共 API（`src/api/`）冻结后只增不改，破坏性变更必须升主版本
- 内部实现可随时重构，外部调用无感知

### 2.2 依赖倒置

- 业务代码依赖 `IMapEngine` 接口，而非具体的 Cesium / OL 实现
- 数据源依赖 `IDataSource` 接口，HTTP / WebSocket / Mock 可互换

### 2.3 渐进增强

- 核心包（`@geomapcore/core`）零依赖，适配器按需引入
- 不强制引入完整 SDK，业务方可只取所需模块

### 2.4 内网安全

- 所有网络请求地址均通过配置注入，无硬编码外网 URL
- 支持请求拦截器，方便注入鉴权 Token、签名等

### 2.5 可观测性

- 完整的日志分级（DEBUG / INFO / WARN / ERROR）
- 关键操作均有性能打点，支持外部 APM 接入
- 数据流可追踪，便于联调


### 4.2 数据流向

```
外部数据源（HTTP / WebSocket / 本地）
    │
    ▼
DataSourceManager
    │ 中间件链（鉴权 → 缓存 → 重试 → 日志）
    ▼
IDataSource（标准数据对象）
    │
    ▼
LayerManager → ILayer → IMapEngine → 渲染到地图
    │
    ▼
EventBus → 业务回调（地图事件 / 数据更新事件）
```

### 4.3 生命周期

```
create(options)
    │
    ├── 初始化引擎（OL 或 Cesium）
    ├── 初始化事件总线
    ├── 初始化数据源管理器
    ├── 初始化图层管理器
    ├── 注册内置插件
    └── 触发 onReady 回调
            │
            ▼
        运行中
            │
            ├── addLayer / removeLayer
            ├── updateDataSource
            ├── on / off 事件监听
            └── 插件调用
            │
            ▼
        destroy()
            │
            ├── 销毁所有图层
            ├── 断开所有数据源连接
            ├── 卸载所有插件
            ├── 移除事件监听
            └── 销毁引擎实例
```

## 5. 类型系统与公共接口

### 5.1 基础地理类型（`map.types.ts`）

```typescript
// ============================================================
// 基础地理坐标类型
// 使用 WGS84 经纬度坐标系作为统一标准
// 内部坐标转换由适配器负责（如 OL 默认 EPSG:3857）
// ============================================================

/** 经纬度坐标 [经度, 纬度] */
export type LngLat = [number, number];

/** 经纬度高度坐标 [经度, 纬度, 高度（米）] */
export type LngLatAlt = [number, number, number];

/** 屏幕像素坐标 [x, y] */
export type PixelCoord = [number, number];

/** 地图视野范围：西南角与东北角经纬度 */
export interface BoundingBox {
  west: number; // 西边界经度
  south: number; // 南边界纬度
  east: number; // 东边界经度
  north: number; // 北边界纬度
}

/** 地图视图状态 */
export interface ViewState {
  center: LngLat; // 当前地图中心经纬度
  zoom: number; // 当前缩放级别（2D）/ 相机高度（3D 时转换）
  rotation?: number; // 旋转角度（弧度，OL 支持，Cesium 通过 heading 实现）
  pitch?: number; // 俯仰角（3D，度），0 为正射，90 为水平
  heading?: number; // 朝向角（3D，度），0 为正北
}

/** flyTo 动画参数 */
export interface FlyToOptions {
  center: LngLat | LngLatAlt; // 目标中心点
  zoom?: number; // 目标缩放级别
  duration?: number; // 动画时长（毫秒），默认 1000
  pitch?: number; // 目标俯仰角（3D）
  heading?: number; // 目标朝向（3D）
  easing?: (t: number) => number; // 自定义缓动函数
}

/** 渲染引擎类型枚举 */
export enum EngineType {
  OpenLayers = 'openlayers', // 二维平面地图
  Cesium = 'cesium', // 三维地球
}

/** SDK 初始化主配置 */
export interface MapCoreOptions {
  /** 挂载容器：DOM 元素或 ID 字符串 */
  container: HTMLElement | string;
  /** 渲染引擎选择 */
  engine: EngineType;
  /** 初始视图状态 */
  initialView?: ViewState;
  /** 数据源全局配置 */
  dataSourceConfig?: DataSourceGlobalConfig;
  /** 调试配置 */
  debug?: DebugConfig;
  /** 内网代理配置 */
  proxy?: ProxyConfig;
  /** 插件列表（初始化时自动注册） */
  plugins?: IPlugin[];
}
```

### 5.2 图层类型（`layer.types.ts`）

```typescript
// ============================================================
// 图层类型系统
// 图层是地图上可独立管理的渲染单元
// 所有图层通过 LayerManager 统一管理，引擎无关
// ============================================================

/** 图层类型枚举 */
export enum LayerType {
  // ---- 底图类 ----
  Tile = 'tile', // 栅格瓦片（XYZ / TMS）
  WMS = 'wms', // OGC WMS 服务
  WMTS = 'wmts', // OGC WMTS 服务
  // ---- 矢量类 ----
  Vector = 'vector', // 矢量要素（GeoJSON / WFS）
  // ---- 数据可视化类 ----
  Heatmap = 'heatmap', // 热力图
  Cluster = 'cluster', // 聚合点图
  // ---- 3D 专属 ----
  Tileset3D = 'tileset3d', // Cesium 3D Tiles
  Terrain = 'terrain', // 地形服务
  CZML = 'czml', // CZML 动态数据（Cesium）
  // ---- 自定义类 ----
  Custom = 'custom', // 业务自定义图层（插件扩展）
}

/** 图层通用基础配置 */
export interface LayerBaseConfig {
  /** 图层唯一标识，业务方自定义，不可重复 */
  id: string;
  /** 图层类型 */
  type: LayerType;
  /** 图层显示名称（可选，供 UI 展示用） */
  name?: string;
  /** 图层可见性，默认 true */
  visible?: boolean;
  /** 图层透明度 0-1，默认 1 */
  opacity?: number;
  /** 图层叠加顺序（z-index），数值越大越靠上 */
  zIndex?: number;
  /** 图层最小可见缩放级别 */
  minZoom?: number;
  /** 图层最大可见缩放级别 */
  maxZoom?: number;
  /** 图层关联数据源 ID（对应 DataSourceManager 中注册的 ID） */
  sourceId?: string;
  /** 业务自定义元数据，SDK 不做处理，原样保存 */
  metadata?: Record<string, unknown>;
}

/** 瓦片图层配置 */
export interface TileLayerConfig extends LayerBaseConfig {
  type: LayerType.Tile;
  /** 瓦片 URL 模板，支持 {x} {y} {z} 占位符 */
  url: string;
  /** 瓦片尺寸，默认 256 */
  tileSize?: number;
  /** 跨域属性 */
  crossOrigin?: string;
}

/** 矢量图层配置 */
export interface VectorLayerConfig extends LayerBaseConfig {
  type: LayerType.Vector;
  /** 样式配置 */
  style?: VectorStyleConfig;
  /** 要素点击是否可选中 */
  selectable?: boolean;
}

/** 矢量要素样式 */
export interface VectorStyleConfig {
  /** 填充颜色（GeoJSON Polygon 等） */
  fillColor?: string;
  /** 描边颜色 */
  strokeColor?: string;
  /** 描边宽度（像素） */
  strokeWidth?: number;
  /** 点要素半径（像素） */
  pointRadius?: number;
  /** 自定义图标 URL（点要素） */
  iconUrl?: string;
  /** 图标缩放比 */
  iconScale?: number;
  /** 文字标注 */
  label?: {
    field: string; // 要素属性字段名
    font?: string; // CSS font 字符串
    color?: string; // 文字颜色
    offsetY?: number; // Y 轴偏移像素
  };
}

/** 所有图层配置的联合类型 */
export type LayerConfig =
  | TileLayerConfig
  | VectorLayerConfig
  | WMSLayerConfig
  | WMTSLayerConfig
  | HeatmapLayerConfig
  | Tileset3DLayerConfig
  | TerrainLayerConfig
  | CZMLLayerConfig
  | CustomLayerConfig;
```

```typescript
/** 数据源类型枚举 */
export enum DataSourceType {
  HTTP = 'http', // 单次 HTTP/HTTPS 请求
  WebSocket = 'websocket', // 长连接 WebSocket
  Static = 'static', // 本地静态数据
  Mock = 'mock', // Mock 调试数据（仅开发/联调使用）

/** 数据源全局配置（所有数据源共享） */
export interface DataSourceGlobalConfig {
  /** 请求超时时间（毫秒），默认 15000 */
  timeout?: number;
  /** 失败重试次数，默认 3 */
  retryCount?: number;
  /** 重试间隔基础时间（毫秒，指数退避），默认 1000 */
  retryDelay?: number;

  cacheEnabled?: boolean;
  /** 缓存有效期（毫秒） */
  cacheTTL?: number;
  /** 全局请求头（如鉴权 Token） */
  headers?: Record<string, string>;
  /** 请求拦截器（发出前修改请求参数） */
  requestInterceptor?: RequestInterceptor;
  /** 响应拦截器（接收后处理响应数据） */
  responseInterceptor?: ResponseInterceptor;
}

/** 请求拦截器类型：返回修改后的 RequestInit */
export type RequestInterceptor = (
  url: string,
  init: RequestInit
) => Promise<{ url: string; init: RequestInit }>;

/** 响应拦截器类型：返回处理后的数据 */
export type ResponseInterceptor = (response: Response, data: unknown) => Promise<unknown>;

  id: string;

  headers?: Record<string, string>;
  /** POST 请求体 */
  body?: Record<string, unknown>;
  /** 轮询间隔（毫秒），为 0 则不轮询 */
  pollInterval?: number;

  format?: 'geojson' | 'json' | 'arraybuffer' | 'text';

  heartbeatInterval?: number;
  /** 心跳消息内容 */
  heartbeatMessage?: string;

  reconnectLimit?: number;
```

### 6.1 IMapEngine 接口定义（`IMapEngine.ts`）

```typescript
// ============================================================
// IMapEngine：地图引擎统一抽象接口
// 所有上层代码只依赖此接口，不依赖具体引擎实现
// OLMapEngine 和 CesiumMapEngine 均需完整实现此接口
// ============================================================

export interface IMapEngine {
  // ---- 生命周期 ----

  /**
   * 初始化引擎，异步完成
   * 包含：挂载 DOM、加载资源、创建地图实例
   */
  init(container: HTMLElement, options: MapCoreOptions): Promise<void>;

  /**
   * 销毁引擎，释放所有资源
   * 调用后该实例不可再使用
   */
  destroy(): void;

  // ---- 视图控制 ----

  /**
   * 设置视图（立即跳转，无动画）
   * @param state 目标视图状态
   */
  setView(state: Partial<ViewState>): void;

  /**
   * 获取当前视图状态
   */
  getView(): ViewState;

  /**
   * 飞行到目标位置（带动画）
   * @param options 飞行参数
   */
  flyTo(options: FlyToOptions): Promise<void>;

  /**
   * 获取当前地图可视范围
   */
  getBounds(): BoundingBox;

  // ---- 坐标转换 ----

  /**
   * 地理坐标 → 屏幕像素坐标
   * @param lngLat 经纬度
   * @returns 像素坐标，若超出视口返回 null
   */
  project(lngLat: LngLat): PixelCoord | null;

  /**
   * 屏幕像素坐标 → 地理坐标
   * @param pixel 像素坐标
   */
  unproject(pixel: PixelCoord): LngLat;

  // ---- 图层操作 ----

  /**
   * 添加图层
   * 具体图层对象由各适配器内部创建
   */
  addLayer(config: LayerConfig): void;

  /**
   * 移除图层
   * @param layerId 图层 ID
   */
  removeLayer(layerId: string): void;

  /**
   * 设置图层可见性
   */
  setLayerVisible(layerId: string, visible: boolean): void;

  /**
   * 设置图层透明度
   */
  setLayerOpacity(layerId: string, opacity: number): void;

  /**
   * 调整图层 z 顺序
   */
  setLayerZIndex(layerId: string, zIndex: number): void;

  /**
   * 更新图层数据（适用于矢量图层）
   * @param layerId 图层 ID
   * @param data 新的 GeoJSON 数据
   */
  updateLayerData(layerId: string, data: GeoJSONFeatureCollection): void;

  // ---- 要素查询 ----

  /**
   * 查询指定像素坐标处的要素
   * @param pixel 像素坐标
   * @param options 查询选项（命中容差、指定图层等）
   */
  queryFeaturesByPixel(pixel: PixelCoord, options?: QueryOptions): GeoJSONFeature[];

  /**
   * 查询指定范围内的要素
   * @param bbox 范围框
   * @param layerIds 限定图层，不传则查所有可见图层
   */
  queryFeaturesByBBox(bbox: BoundingBox, layerIds?: string[]): GeoJSONFeature[];

  // ---- 原生引擎访问（逃生舱口）----

  /**
   * 获取底层引擎实例（OL Map 或 Cesium Viewer）
   * 用于访问 SDK 未封装的底层 API
   * 注意：直接操作底层实例可能导致 SDK 状态不同步
   * @returns OL 返回 ol/Map 实例，Cesium 返回 Cesium.Viewer 实例
   */
  getNativeInstance(): unknown;
}
```

## 9. 事件系统

### 9.1 EventBus（事件总线）

```typescript
// ============================================================
// EventBus：统一事件总线
// 实现发布/订阅模式，解耦各模块间通信
//
// 特性：
//   - 类型安全：通过泛型约束事件载荷类型
//   - 支持一次性监听（once）
//   - 支持通配符监听（监听所有事件，调试用）
//   - 支持优先级（高优先级处理器先执行）
//   - 异步事件处理（emit 不阻塞，处理器可以是 async）
// ============================================================

export class EventBus {
  private handlers: Map<string, EventHandler[]> = new Map();
  private wildcardHandlers: EventHandler[] = [];

  /**
   * 订阅事件
   * @param event 事件名（使用 MapEvents 常量）
   * @param handler 处理函数
   * @param priority 优先级，数值越大越先执行，默认 0
   * @returns 取消订阅函数
   */
  on<T = unknown>(
    event: string,
    handler: (payload: T) => void | Promise<void>,
    priority?: number
  ): () => void;

  /**
   * 订阅一次性事件（触发后自动取消）
   */
  once<T = unknown>(event: string, handler: (payload: T) => void): void;

  /**
   * 订阅所有事件（通配符，仅调试建议使用）
   */
  onAll(handler: (event: string, payload: unknown) => void): () => void;

  /**
   * 触发事件
   * @param event 事件名
   * @param payload 事件载荷
   */
  emit<T = unknown>(event: string, payload: T): void;

  /**
   * 取消订阅
   */
  off(event: string, handler: Function): void;

  /**
   * 清除指定事件的所有订阅（销毁时使用）
   */
  clear(event?: string): void;
}
```

### 10.1 IPlugin 接口（`IPlugin.ts`）

```typescript
// 插件是 SDK 功能的扩展单元，遵循统一生命周期

// 设计意图：
//   - SDK 保持核心精简，特定场景功能通过插件实现
```

### 11.3 跨端通信 Bridge（Qt / 移动端 / iframe）

```typescript
// ============================================================
// BridgeFactory：跨端通信桥工厂
// 自动检测宿主环境，返回对应的 Bridge 实现
// 统一通信协议：JSON-RPC 2.0 风格
// ============================================================

export class BridgeFactory {
  static detect(): IBridge {
    // Qt WebChannel
    if (typeof window.qt !== 'undefined' && window.qt.webChannelTransport) {
      return new QtWebChannelBridge();
    }
    // Android JSBridge
    if (typeof window.AndroidBridge !== 'undefined') {
      return new AndroidBridge();
    }
    // iOS WKWebView
    if (window.webkit?.messageHandlers?.mapBridge) {
      return new IOSBridge();
    }
    // iframe / opener 通信
    if (window.parent !== window || window.opener) {
      return new PostMessageBridge();
    }
    // 纯浏览器环境，使用 NullBridge（不做任何跨端通信）
    return new NullBridge();
  }
}

// ---- 通信协议定义 ----
interface BridgeMessage {
  /** 消息唯一 ID（用于请求/响应配对） */
  id: string;
  /** 方法名：sdk.mapClick / sdk.layerUpdate 等 */
  method: string;
  /** 参数 */
  params?: unknown;
  /** 响应结果（仅响应消息有） */
  result?: unknown;
  /** 错误信息（仅错误响应有） */
  error?: { code: number; message: string };
}

// ---- Qt WebChannel Bridge ----
class QtWebChannelBridge implements IBridge {
  private channel: QWebChannel;

  async send(method: string, params: unknown): Promise<unknown> {
    // 通过 QWebChannel 调用 Qt 侧注册的方法
    // 返回 Promise，等待 Qt 侧回调
  }

  receive(handler: (method: string, params: unknown) => void): void {
    // 监听 Qt 侧主动推送的消息
  }
}

// ---- PostMessage Bridge（iframe / WebView）----
class PostMessageBridge implements IBridge {
  private pendingRequests: Map<string, PromiseResolver> = new Map();

  send(method: string, params: unknown): Promise<unknown> {
    const id = crypto.randomUUID();
    const message: BridgeMessage = { id, method, params };
    // 发往 parent 或 opener 窗口
    const target = window.parent !== window ? window.parent : window.opener;
    target.postMessage(message, '*'); // 内网场景可配置具体 origin
    // 返回 Promise，等待对应 id 的响应消息
    return new Promise((resolve, reject) => {
      this.pendingRequests.set(id, { resolve, reject });
      // 超时处理
    });
  }

  receive(handler: (method: string, params: unknown) => void): void {
    window.addEventListener('message', (e: MessageEvent) => {
      const msg = e.data as BridgeMessage;
      if (msg.id && this.pendingRequests.has(msg.id)) {
        // 响应消息，resolve 对应 Promise
      } else if (msg.method) {
        // 主动推送消息，调用 handler
        handler(msg.method, msg.params);
      }
    });
  }
}
```

## 16. 扩展点汇总

| 扩展点         | 扩展方式                                                               | 使用场景                         |
| -------------- | ---------------------------------------------------------------------- | -------------------------------- |
| 自定义图层类型 | 实现 `ILayer`，通过 `LayerManager.registerLayerType()` 注册            | 特殊渲染效果、军标符号图层       |
| 自定义数据源   | 实现 `IDataSource`，通过 `DataSourceManager.registerSourceType()` 注册 | 私有协议数据接入                 |
| 请求拦截       | `dataSourceConfig.requestInterceptor`                                  | 注入鉴权 Token、URL 改写、加密   |
| 响应拦截       | `dataSourceConfig.responseInterceptor`                                 | 数据格式转换、解密、过滤         |
| 中间件         | `DataSourceManager.addMiddleware()`                                    | 通用逻辑（缓存、重试、限流）     |
| 插件           | 实现 `IPlugin`，通过 `map.use()` 注册                                  | 业务功能扩展（测量、绘图、回放） |
| 事件监听       | `map.on(event, handler)`                                               | 响应地图/数据/系统事件           |
| 日志处理       | `logger.addHandler(handler)`                                           | 接入内网日志收集系统             |
| 性能上报       | `performanceMonitor.addReporter(reporter)`                             | 接入内网 APM 系统                |
| 跨端通信       | 实现 `IBridge`，通过 `BridgeFactory` 注入                              | 自定义 Native 通信协议           |
| 引擎底层访问   | `engine.getNativeInstance()`                                           | 访问 SDK 未封装的底层 API        |
